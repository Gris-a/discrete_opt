## Отчёт

### Greedy (Жадный алгоритм)
Базовое решение, основанное на выборе множества, покрывающего наиболльшее число ещё не покрытых элементов (`solve_greedy`).

| sc_157_0   | 102100 | 3 / 5 |
| sc_330_0   | 30     | 0 / 5 |
| sc_1000_11 | 157    | 3 / 5 |
| sc_5000_1  | 32     | 3 / 5 |
| sc_10000_5 | 69     | 3 / 5 |
| sc_10000_2 | 182    | 3 / 5 |
| Итого      |        | 15 / 30 |

Почти везде пробили первые пороги, попробуем улучшить жадное решение.

### Simulated Annealing
На каждом шаге выкидываем случайно 3 множества и восстанавливаем покрытие жадно, с некоторой вероятностью идем к худшему решению чтобы не застревать в локальных минимумах (`solve_annealing`).

| sc_157_0   | 99900  | 3 / 5 |
| sc_330_0   | 24     | 5 / 5 |
| sc_1000_11 | 151    | 3 / 5 |
| sc_5000_1  | 31     | 5 / 5 |
| sc_10000_5 | 64     | 5 / 5 |
| sc_10000_2 | 175    | 3 / 5 |
| Итого      |        | 24 / 30 |

Как видно алгоритм оказался очень удачным.
