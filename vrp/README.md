```bash
./run
```

## Тестирование
чтобы прогнать тесты самому и увидеть полный ответ после сборки
```bash
./bin/vrp tests/<test_file>
```

## Постановка
Есть `n` покупателей, `v` машин, 1 склад. Вместимость каждой машины `c` единиц товара, каждому покупателю требуется `d_i` единиц товара. Нужно распределить каждой машине покупателей, которых она обслужит и минимизировать сумму длин путей со склада до покупателей и обратно.

## Отчёт

### Greedy
`assign_routes_nearest` — сначала строится порядок клиентов по жадному приближению (каждого следующего клиента выбираем ближайшим к текущему положению).
`assign_routes_sweep` — клиенты сортируются по углу от склада и распределяются по машинам по возрастанию угла. Это помогает группировать географически близкие доставки.

Одним из двух методов сортируются все доставки, затем клиенты распределяются по маршрутам машин так, чтобы вместимость каждой машины не превышалась. После разбиения клиентов по машинам проходит оптимизация маршрутов каждой машины с помощью солвера `TSP` (сейчас просто 2opt). Итоговая стоимость считается как сумма длин маршрутов всех машин с возвратом в начало склада, выбирается результат при лучшем из двух возможных распределений по машинам (`vrp_greedy`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| vrp_16_3_1   | 285.5   | 3 / 5 |
| vrp_26_8_1   | 707.8   | 3 / 5 |
| vrp_51_5_1   | 593.0   | 3 / 5 |
| vrp_101_10_1 | 998.0   | 3 / 5 |
| vrp_200_16_1 | 1596.2  | 3 / 5 |
| vrp_421_41_1 | 2153.8  | 3 / 5 |
| Итого        |         | 18 / 25 |

Результат довольно неплохой, местами даже близко к второму порогу.
