## Отчёт

### Greedy (Жадный алгоритм)
Базовое решение, основанное на сортировке предметов по удельной стоимости ($v_i/w_i$), берём по очереди начиная с самых ценных (`solve_greedy`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| ks_30_0    | 90000   | 0 / 5   |
| ks_50_0    | 141956  | 3 / 5   |
| ks_200_0   | 100062  | 3 / 5   |
| ks_400_0   | 3966813 | 3 / 5   |
| ks_1000_0  | 109869  | 3 / 5   |
| ks_10000_0 | 1099870 | 3 / 5   |
| Итого      |         | 15 / 30 |

Почти пробил первые пороги, но этого недостаточно. Попробуем его улучшить.

### Hybrid (DP + Greedy)
Гибридный подход. На префиксе из первых $k$ элементов выполняется точное динамическое программирование (DP). Оставшаяся вместимость заполняется жадно. Параметр $k$ подбирается из соображений что алгоритм занимает $k \cdot W$ памяти, а я хочу чтобы он занимал менее 1Гб, поэтому $k = \min(n, 2^{30} / W)$ (`solve_hybrid`). (идею по ограничению k взял у кого-то)

Видно что на маленьких тестах находя точное решение получаем пробитие вторых порогов, но на больших этого не достаточно.

### Branch and Bound (Метод ветвей и границ)
Использует обход дерева решений в глубину с отсечением веток с помощью верхней оценки (релаксация). Сначала предметы сортируются чтобы потом было просто оценивать сверху, на каждом шаге если мы не можем теоретически получить лучшую оценку то мы возвращаемся, иначе делаем отсечение: берем или не берем текущий предмет.
