## Отчёт

### Greedy (Жадный алгоритм)
Самое простое решение, выбирается вершина и ей присваивается цвет минимальный возможный не конфликтующий с соседями (`greedy`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| gc_50_3    | 9    | 0 / 5  |
| gc_70_7    | 23   | 0 / 5  |
| gc_100_5   | 23   | 0 / 5  |
| gc_250_9   | 100  | 0 / 5  |
| gc_500_1   | 20   | 0 / 5  |
| gc_1000_5  | 127  | 0 / 5  |
| Итого      |      | 0 / 30 |

Видно что алгоритм совсем не справляется, хотя как-будто близок к первым порогам, поэтому воспользуемся эвристиками.

### DSatur
Также жадный алгоритм, но теперь вершины выбираются не в случайном порядке, а ту вершину, у которой число различных цветов у соседей наибольшее (`dsatur`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| gc_50_3    | 7    | 3 / 5  |
| gc_70_7    | 21   | 0 / 5  |
| gc_100_5   | 19   | 3 / 5  |
| gc_250_9   | 91   | 3 / 5  |
| gc_500_1   | 16   | 5 / 5  |
| gc_1000_5  | 115  | 3 / 5  |
| Итого      |      | 17 / 30 |

Видим что почти все пороги пробили,поэтому можно ещё попробовать эвристики.

### RLF
Алгоритм пытается сформировать для текущего цвета максимально возможное независимое подмножество вершин. Таким образом на ранних итерациях мы можем покрасить в один цвет очень много вершин (`rlf`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| gc_50_3    | 7    | 3 / 5  |
| gc_70_7    | 18   | 3 / 5  |
| gc_100_5   | 17   | 3 / 5  |
| gc_250_9   | 84   | 3 / 5  |
| gc_500_1   | 15   | 5 / 5  |
| gc_1000_5  | 107  | 3 / 5  |
| Итого      |      | 20 / 30 |

Теперь пробились все первые пороги и близки вторые, поэтому можно попробовать искать решение локальным поиском.

### Kempe Chains
Перестановки Кемпе берут готовое решение и пытаются улучшить. Берется вершина максимального цвета и мы пытаемся её перекрасить, если есть одна вершина-сосед с конкретным цветом, то строится двуцветная цепь начиная с этих вершин и потом в цепи меняется покраска (`kempe`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| gc_50_3    | 7    | 3 / 5  |
| gc_70_7    | 18   | 3 / 5  |
| gc_100_5   | 17   | 3 / 5  |
| gc_250_9   | 84   | 3 / 5  |
| gc_500_1   | 15   | 5 / 5  |
| gc_1000_5  | 107  | 3 / 5  |
| Итого      |      | 20 / 30 |

Как видно никаких улучшений не получили, возможно алгоритм RLF находит глубокий локальный минимум, попробуем из него выбраться.

### Simulatend Annealing
На каждой итерации пытаются изменить вершины также максимального цвета. Изначально им присваиваются случайные цвета, вычисляется число конфликтов. На каждом шаге выбирается вершина с конфликтами и ей подбирается новый цвет. Если конфликтов стало больше то с некоторой вероятностью мы берем этот вариант, что помогает вылезти из локального минимума (`simulated_annealing`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| gc_50_3    | 6    | 5 / 5  |
| gc_70_7    | 18   | 3 / 5  |
| gc_100_5   | 16   | 5 / 5  |
| gc_250_9   | 78   | 5 / 5  |
| gc_500_1   | 15   | 5 / 5  |
| gc_1000_5  | 107  | 3 / 5  |
| Итого      |      | 26 / 30 |
