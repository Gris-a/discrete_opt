## Сборка и тестирование
```bash
./run
```

## Тестирование
чтобы прогнать тесты самому и увидеть полный ответ после сборки
```bash
./bin/tsp tests/<test_file>
```

## Отчёт

### Greedy
Простой жадный алгоритм, просто берет ближайшего и идёт к нему и так пока не обойдёт всех (`tsp_greedy`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| tsp_51_1    | 506.4       | 0 / 5  |
| tsp_100_3   | 25138.8     | 0 / 5  |
| tsp_200_2   | 36226.2     | 0 / 5  |
| tsp_574_1   | 47054.9     | 0 / 5  |
| tsp_1889_1  | 391470.4    | 0 / 5  |
| tsp_33810_1 | 78478867.0  | 3 / 5  |
| Итого       |             | 3 / 30 |

Это решение очень плохое, попробуем улучшить.

### 2opt
Алгоритм берет пары ребер и меняет их на диагональные, и так пока не улучшит или не получит ограничение по времени (`tsp_2opt`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| tsp_51_1    | 440.2       | 3 / 5  |
| tsp_100_3   | 21746.2     | 3 / 5  |
| tsp_200_2   | 31499.5     | 3 / 5  |
| tsp_574_1   | 39538.7     | 3 / 5  |
| tsp_1889_1  | 341010.7    | 3 / 5  |
| tsp_33810_1 | 77529276.9  | 3 / 5  |
| Итого       |             | 18 / 30 |

Получилось везде пробить первые пороги, местами ответ близок ко вторым.

### Simulated Annealing

Имитация отжига, на каждой итерации сначала делается случайный 4opt, потом решение улучшается на 2opt. Чтобы не думать о размерах задачи решение ищется 1 минуту, можно сделать быстрее (`tsp_annealing`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| tsp_51_1    | 429.5       | 5 / 5  |
| tsp_100_3   | 20750.8     | 5 / 5  |
| tsp_200_2   | 29747.7     | 5 / 5  |
| tsp_574_1   | 37841.1     | 3 / 5  |
| tsp_1889_1  | 327954.4    | 3 / 5  |
| tsp_33810_1 | 77916776.3  | 3 / 5  |
| Итого       |             | 24 / 30 |

Решение оказалось удивительно рабочим, пробило 3 больших порога.
