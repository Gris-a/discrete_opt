## Отчёт
### Greedy
Для каждого клиента выбирается допустимый объект с минимальной суммы расстояния до клиента и стоимости открытия (если объект ещё закрыт). После назначения клиента объект помечается как открытый (если это было нужно) и его использованная ёмкость увеличивается (`facility_greedy`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| fl_25_2   | 3269821.3   | 5 / 5 |
| fl_100_1  | 47302882.4  | 0 / 5 |
| fl_200_7  | 7765704.5   | 0 / 5 |
| fl_500_7  | 83975185.4  | 0 / 5 |
| fl_1000_2 | 12076465.4  | 0 / 5 |
| fl_2000_2 | 12479411.0  | 0 / 5 |
| Итого     |             | 5 / 30 |

это очень плохо, попробуем найти эвристику

### Regret2
Для каждого клиента, еще не привязанного к фабрике, ищем лучшую фабрику и вторую лучшую фабрику, вычисляем `regret` - насколько мы ухудшим ответ если возьмём не лучшую фабрику а вторую, выбераем клиента с максимальным значением `regret` и назначаем его на его лучшую фабрику (`facility_regret`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| fl_25_2   | 3269821.3   | 5 / 5 |
| fl_100_1  | 33456813.4  | 0 / 5 |
| fl_200_7  | 5642008.7   | 0 / 5 |
| fl_500_7  | 35186089.1  | 0 / 5 |
| fl_1000_2 | 10020082.8  | 0 / 5 |
| fl_2000_2 | 8472301.5   | 3 / 5 |
| Итого     |             | 8 / 30 |

Получили существенный прирост, местами выглядит близко к порогам. Попробуем локальный поиск.

### Simulated Annealing

Введём 3 возможные мутации: клиент меняет фабрику, 2 фабрики меняются клиентами, закрывается одна фабрика (`facility_annealing`).

| тест | значение | порог |
|:----:|:--------:|:-----:|
| fl_25_2   | 3269821.3   | 5 / 5 |
| fl_100_1  | 28023570.0  | 0 / 5 |
| fl_200_7  | 4961586.4   | 3 / 5 |
| fl_500_7  | 32516623.4  | 0 / 5 |
| fl_1000_2 | 9450731.2   | 3 / 5 |
| fl_2000_2 | 8050929.0   | 3 / 5 |
| Итого     |             | 14 / 30 |

Видим улучшение, но этого не достаточно. Я забыл что фабрики можно открывать, поэтому решение можно ещё улучшить.
